# GOLD ブレイクアウト×モメンタム継続 ハイブリッド戦略 設計書（Draft）

## 0. コンセプト

**「走り出すと続く」GOLD の特性を活かし、レンジブレイクで乗り、モメンタムが続く限りホールドする。**

既存 EA（ScalpImpulseRetraceEA）の Impulse→Retrace 型とは完全に別戦略。
FX 用 EA とは独立した新規 EA として設計する。

---

## 1. 戦略の全体フロー

```
Asian Session  →  London/NY Open  →  Breakout  →  Hold  →  Exit
[レンジ形成]     [ブレイク監視]     [エントリー]  [継続判定]  [決済]
```

### フェーズ概要

| Phase | 内容 | 時間帯（サーバー時間目安） |
|-------|------|--------------------------|
| 1. Range Build | Asian セッションの High/Low を記録 | 00:00–07:00 |
| 2. Breakout Watch | London open 以降、レンジ外への Close を監視 | 07:00–20:00 |
| 3. Entry | ブレイク確認 → エントリー | ブレイク発生時 |
| 4. Hold / Trail | モメンタム継続中はトレーリングでホールド | エントリー後 |
| 5. Exit | トレーリング SL ヒット or セッション終了 | — |

---

## 2. Phase 1: Range Build（レンジ形成）

### 2.1 レンジの定義

- **期間**: Asian セッション（サーバー時間 00:00〜07:00）
- **足**: M15 を使用（ノイズ除去のため M1 は使わない）
- **RangeHigh**: 期間内の最高値
- **RangeLow**: 期間内の最安値
- **RangeWidth**: RangeHigh - RangeLow

### 2.2 レンジ品質フィルタ

レンジが狭すぎる or 広すぎる場合はトレードしない:

| 条件 | 判定 | 理由 |
|------|------|------|
| RangeWidth < ATR(H1, 14) × 0.3 | Skip | 極端に狭い → 偽ブレイク多発 |
| RangeWidth > ATR(H1, 14) × 1.5 | Skip | 既にトレンド発生済み → 出遅れ |

### 2.3 補足: サーバー時間の扱い

XMTrading のサーバー時間は GMT+2（夏時間 GMT+3）。
Asian セッション = 東京 09:00–16:00 = サーバー 02:00–09:00 が実態に近い。
→ **Input パラメータで RangeStartHour / RangeEndHour を設定可能にする**（ブローカー移行対応）。

---

## 3. Phase 2: Breakout Watch（ブレイク監視）

### 3.1 監視足

- **M15**: ブレイク判定（ノイズ回避）
- M5 も候補だが、GOLD の M5 はスパイクが多く偽ブレイクリスクが高い → M15 推奨

### 3.2 ブレイク条件

**M15 足の Close が RangeHigh / RangeLow を超えた時点でブレイク成立。**

追加フィルタ:
| フィルタ | 条件 | 目的 |
|----------|------|------|
| Body比率 | ブレイク足の Body ≥ Range の 50% of bar range | ヒゲだけのブレイクを除外 |
| 超過距離 | Close が Range 境界から Spread × 2 以上離れている | スプレッド分のノイズを除外 |

### 3.3 ブレイク方向

- Close > RangeHigh → **Long ブレイク**
- Close < RangeLow → **Short ブレイク**

### 3.4 1日1方向ルール

- 最初のブレイク方向のみ有効。
- 例: Long ブレイクで SL → その日は再エントリーしない（ダマシ後の逆ブレイクは見送り）。
- 理由: 両方向でエントリーすると往復ビンタのリスク。

---

## 4. Phase 3: Entry（エントリー）

### 4.1 エントリータイミング

ブレイク足の Close 確定時に **成行（Market）** でエントリー。

- Limit 注文は使わない（ブレイク後に戻ってこない前提）
- スリッページ許容: MaxSlippage = 5 pts（現行 GOLD 設定と同じ）

### 4.2 ポジションサイズ

- SL 距離からロット計算（リスク固定型）
- 1トレードリスク = 口座残高の N%（Input: RiskPercent, default 1.0%）

### 4.3 SL（損切り）

**SL = レンジの反対側 - ATR(M15) × 0.3**

- Long の場合: SL = RangeLow - ATR(M15) × 0.3
- Short の場合: SL = RangeHigh + ATR(M15) × 0.3
- 理由: レンジ反対側 + バッファ。ブレイクが本物ならここまで戻らない。

---

## 5. Phase 4: Hold / Trail（モメンタム継続判定）

### 5.1 トレーリングストップ

ブレイク後のホールドはトレーリング SL で管理。

**方式: ATR ベース・トレーリング**

```
TrailDistance = ATR(M15, 14) × TrailMult
TrailMult = 1.5（Input パラメータ）

Long の場合:
  NewSL = CurrentPrice - TrailDistance
  SL = max(SL, NewSL)  // SL は切り上げのみ

Short の場合:
  NewSL = CurrentPrice + TrailDistance
  SL = min(SL, NewSL)  // SL は切り下げのみ
```

### 5.2 建値移動（Breakeven）

- 含み益が ATR(M15) × 1.0 以上になったら SL をエントリー価格 + Spread に移動
- 理由: ブレイクが戻った場合の損失をゼロにする

### 5.3 トレーリング更新タイミング

- M15 足の確定時に更新（ティックごとではない）
- 理由: M1/ティック更新だとノイズで SL が近づきすぎる

---

## 6. Phase 5: Exit（決済）

### 6.1 決済条件（どれか1つで決済）

| # | 条件 | 説明 |
|---|------|------|
| 1 | **トレーリング SL ヒット** | メインの出口。モメンタムが止まった証拠 |
| 2 | **セッション終了** | NY Close 付近（サーバー時間 23:00）で強制決済 |
| 3 | **逆方向ブレイク** | M15 で逆方向のレンジブレイク級の動き（ATR×2超の逆行足） |

### 6.2 持ち越し禁止

- 原則として日またぎ持ち越しはしない
- GOLD はスワップコストが高く、ギャップリスクもある
- NY Close（サーバー 23:00）までに決済

---

## 7. リスク管理

### 7.1 1日の制限

| 項目 | 値 | 理由 |
|------|-----|------|
| MaxTradesPerDay | 1 | 1日1方向ルールと整合 |
| MaxDailyLoss | 口座残高の 3% | 連敗時の資金防衛 |

### 7.2 スプレッドフィルタ

- エントリー時のスプレッドが Median(15min) × 3.0 を超えたらエントリーしない
- 指標発表前後のワイドスプレッド回避

---

## 8. 上位足フィルタ（オプション）

### 8.1 H1 トレンド方向フィルタ

- EMA50(H1) の傾きでトレンド方向を判定
- ブレイク方向と H1 トレンドが一致する場合のみエントリー
- **初期実装では OFF**（まずブレイクアウト単体の性能を検証）
- 理由: フィルタを入れすぎるとトレード数が激減し、検証困難になる

### 8.2 重要指標フィルタ（将来拡張）

- FOMC, NFP, CPI 等の前後 N 分はトレード禁止
- MQL5 では経済カレンダー API あり → 将来対応可能

---

## 9. 想定パフォーマンス特性

| 項目 | 想定 |
|------|------|
| トレード頻度 | 1日0〜1回（レンジ形成日のみ） |
| 勝率 | 40〜50%（ブレイクアウト系の一般的な水準） |
| リスクリワード | 1:2〜1:5（トレーリングで利を伸ばす） |
| 勝ちパターン | London open でブレイク → NY まで続伸 |
| 負けパターン | 偽ブレイク → SL（レンジ反対側） |
| 期待値の源泉 | 低勝率を RR で補う。1回の勝ちで2〜3回の負けをカバー |

---

## 10. 既存 EA との関係

- **完全に独立した新規 EA** として実装
- ScalpImpulseRetraceEA のコードベースは流用しない
- MarketProfile / RiskManager 等の共通ユーティリティも新規作成
- 理由: 戦略思想が根本的に異なるため、既存コードの改修は複雑化を招く

---

## 11. 実装ステップ（案）

1. **基本骨格**: レンジ検出 → ブレイク判定 → エントリー → SL/TP
2. **トレーリング**: ATR ベーストレーリング追加
3. **フィルタ追加**: レンジ品質、スプレッド、セッション制限
4. **ログ**: トレード記録（ブレイク幅、保持時間、RR 等）
5. **バックテスト**: 直近6ヶ月の XAUUSD M15 で検証
6. **最適化**: パラメータ調整（TrailMult, レンジ時間帯, SL 倍率等）

---

## 12. Input パラメータ一覧（案）

| パラメータ | 型 | Default | 説明 |
|-----------|-----|---------|------|
| RangeStartHour | int | 0 | レンジ開始（サーバー時間） |
| RangeEndHour | int | 7 | レンジ終了（サーバー時間） |
| TradeStartHour | int | 7 | トレード開始 |
| TradeEndHour | int | 20 | 新規エントリー締切 |
| ForceCloseHour | int | 23 | 強制決済時刻 |
| RiskPercent | double | 1.0 | 1トレードリスク（%） |
| TrailMult | double | 1.5 | トレーリング距離 = ATR × TrailMult |
| BreakevenATRMult | double | 1.0 | 建値移動トリガー = ATR × この値 |
| MinRangeATRMult | double | 0.3 | 最小レンジ幅 = ATR(H1) × この値 |
| MaxRangeATRMult | double | 1.5 | 最大レンジ幅 = ATR(H1) × この値 |
| SLBufferATRMult | double | 0.3 | SL バッファ = ATR(M15) × この値 |
| MaxSpreadMult | double | 3.0 | スプレッドフィルタ倍率 |
| MaxTradesPerDay | int | 1 | 1日最大トレード数 |
| MaxDailyLossPct | double | 3.0 | 1日最大損失（%） |
| UseH1TrendFilter | bool | false | H1 トレンドフィルタ ON/OFF |
